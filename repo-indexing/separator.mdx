# Separator Component

## Overview

### Purpose

Use this component to generate separator lines that visually divide content sections, creating clear boundaries between different areas of your interface.

### When to Use

- **Content Section Division**: Separate distinct content areas within cards, forms, or layouts
- **Menu Item Separation**: Create visual breaks between groups of related menu items
- **List Organization**: Distinguish between different categories or types of list items
- **Form Field Grouping**: Visually separate logical groups of form fields
- **Navigation Elements**: Divide navigation sections or breadcrumb elements
- **Layout Structure**: Add subtle visual breaks in complex layouts without heavy borders

### Architecture

**No Coupling**: This is a standalone component that can be used independently without requiring any parent components or shared context.

## Components

### Separator

**Purpose:** Creates a visual separator line with configurable orientation and styling that enhances content organization and visual hierarchy.

**Interface:**

```typescript
interface SeparatorProps
  extends React.ComponentProps<typeof SeparatorPrimitive.Root> {
  className?: string; // Additional CSS classes for custom styling
  orientation?: "horizontal" | "vertical"; // Direction of the separator line (default: "horizontal")
  decorative?: boolean; // Whether separator is decorative only (default: true)
}
```

**Built on:** [@radix-ui/react-separator](https://www.radix-ui.com/docs/primitives/components/separator) primitive for accessible separator implementation.

**Usage Example:**

```tsx
import { Separator } from "@/components/ui/separator";

function Example() {
  return (
    <div className="space-y-4">
      <div>Content above</div>
      <Separator />
      <div>Content below</div>
    </div>
  );
}
```

## Best Practices

### Usage Guidelines

- **Spacing Control**: Always apply appropriate margin/padding around separators using classes like `my-4`, `mx-2` to ensure proper visual spacing
- **Semantic Purpose**: Use `decorative={false}` when the separator has semantic meaning for screen readers, otherwise keep the default `decorative={true}`
- **Visual Hierarchy**: Use separators sparingly to avoid visual clutter - they should enhance content organization, not overwhelm it
- **Responsive Design**: Consider how separators behave on different screen sizes, especially vertical separators in mobile layouts

### Accessibility

- **Screen Reader Support**: The component uses ARIA separator role when `decorative={false}` for semantic separators
- **Decorative Mode**: Default `decorative={true}` hides the separator from assistive technologies when it's purely visual
- **Orientation Awareness**: Proper orientation attributes ensure correct interpretation by accessibility tools
- **Focus Management**: Separators are not focusable elements and don't interfere with keyboard navigation

### Performance

- **Lightweight Rendering**: Minimal DOM footprint with efficient CSS-based styling
- **No JavaScript Logic**: Pure CSS implementation ensures optimal performance
- **Utility Classes**: Uses Tailwind utility classes for efficient CSS bundling and reuse
